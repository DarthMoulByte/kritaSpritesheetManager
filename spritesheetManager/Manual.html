Spritesheet manager: imports, exports and merges spritesheets


Why use it

Most pixel art games use spritesheets for their animations, but manually exporting from the animation timeline is a pain, effectively destroys your animation if you do it on the same file, and with every new modification you need to start all over again.

This plugin is threefold:
- export: it allows you to easily export the active animation as a spritesheet.
- merge: in case you have several animations on different layers, or just want to make a spritesheet from previously-exported images, it lets you merge external image files afterwards, without needing to necessarily import them into krita.
- import: and you can also import a spritesheet (or part of one) to the animation timeline.


How to use it

Exporting a single animation

Tools > Scripts > SpritesheetManager > Export Spritesheet will export the animation from the currently selected document's animation timeline.
Export direction is horizontal then vertical. For example, in a 3*2 grid, frame numbers are as follows:
1, 2, 3
4, 5, 6

Options:
export name: if you type in "exportName", the name of the individual exported frames will be exportName_000.png, exportName_001.png, etc., and the spritesheet will be called exportName.png . If your name contains a comma, anything after the first comma will be ignored, and the file will still export as a png.
export directory: select the folder in which the spritesheet will be exported (the frames will be exported in a subfolder of the export directory called exportName_sprites)
rows: select the number of rows. If 0, the script will guess it using the number of columns.
columns: select the number of columns. If 0, the script will guess it using the number of rows (it might change the number of rows to respect the horizontal priority)
If both rows and columns are set to 0, the default value will be used for both (I'm torn between the default being one row or a square-ish image, I'd love to hear what other people think about this)
start and end: the start and end of the animation to export to spritesheet. Anything outside those values will not be considered.
If end is set to 0, it will use the default value for both (that is, the Start/End frames of the document (visible in the Animation widget); as such, it may be easier to set them before exporting by right clicking on the timeline and Set Start/End Frame, then leave it at 0 in the export dialog)
remove temporary folder: whether to remove the folder containing the individual exported sprites once finished. You'll usually want this on, but you do you.
